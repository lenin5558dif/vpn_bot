# Instructions for AI agents working on VPN_TG_APP

## Do
- Keep server safety: никогда не выкладывайте ключи WireGuard, BOT_TOKEN, пароли и содержимое `.env` в ответы.
- При деплое на сервер (`/root/VPN_TG_APP`): используйте уже настроенные systemd сервисы (`vpn-backend`, `vpn-bot`); после правок перезапускайте их через `systemctl restart`.
- Backend/WireGuard:
  - Серверный адрес подсети зарезервирован (10.10.0.1/24 по умолчанию); клиенты должны получать адреса, отличные от сервера.
  - Бан peer: должен удалять peer из wg и из БД (+ конфиг).
  - Disable peer: сохраняет peer, но чистит AllowedIPs.
  - Лимит скорости (`apply_speed_limit`) сейчас заглушка — при изменении не ломайте текущую заглушку без необходимости.
- Бот:
  - Админ-меню доступно только ID из `ADMIN_IDS`.
  - Кнопки «Отключить/Активировать/Забанить» должны вызывать PATCH на `/peers/{id}` со статусами `disabled/active/banned`.
  - При выдаче конфига отправляется файл `VPN_<имя_фамилия_транслитом>.conf`.
- Сохраняйте настройки и пути:
  - Рабочая директория на сервере: `/root/VPN_TG_APP`.
  - Виртуальное окружение: `/root/VPN_TG_APP/.venv`.
  - WG конфиг: `/etc/wireguard/wg0.conf`; ключи `/etc/wireguard/server_private.key` и `/etc/wireguard/server_public.key`.
  - Sysctl forward включён в `/etc/sysctl.d/99-wg-forward.conf`.
- При изменении кода на сервере перезапускайте сервисы и проверяйте логи (`journalctl -u vpn-backend -f`, `journalctl -u vpn-bot -f`).
- Тесты: локально `PYTHONPATH=. pytest -q` (минимальный health-check).

## Don’t
- Не публикуйте секреты (.env, ключи wg, токены) в сообщениях/логи вне сервера.
- Не меняйте серверный адрес WG подсети на адрес клиента.
- Не отключайте MASQUERADE/forward правила на wg0 без явной задачи.
- Не запускайте «опасные» команды (reset, rm -rf) в /root вне проекта.
- Не меняйте протоколы взаимодействия бота/бэкенда (эндпоинты, статус-коды) без потребности.

## Tips
- Если кнопки в боте не работают, проверьте, что callback данные обрабатываются соответствующим хендлером и есть ответы `callback.answer(...)`.
- При странных сетевых проблемах у клиента: убедитесь, что порт 51820/udp открыт, peer получил адрес ≠ 10.10.0.1, и в wg0 присутствует MASQUERADE на eth0.
- При добавлении новой логики — обновляйте DOCS.md и перезапускайте сервисы.
